There are two kinds of alerts: those which are emited when raised and those which are emited immediatly (one shot alerts).

The first table display alerts of the first kind, including a cancellation timestamp if the alert as been canceled (cancellation is not immediatly effective to avoid alerts flip/flop).

The second table display every alert (raised or not) along with its emission timestamp. Note that alerts with a human recipient (e.g. e-mail) are aggregated and throttled and therefore may be actually sent to their recipient a few time after if other alerts have already been emited recently.

Alerts rules

For every emited alert, rules are evaluated from top to bottom. If several rules match an alert, it is emited several times, one per matching rule. If a stop rule matches an alert, next rules are not evaluated regardless if they match the alert or not.
